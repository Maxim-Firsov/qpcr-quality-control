# RESULTS

This document summarizes current reproducibility and runtime measurements for the repository baseline after adding public RDML fixtures.

## 1. Benchmark Environment

- OS: Windows 11 (`10.0.26200`)
- Python: `3.13.6`
- CPU: `AMD64 Family 25 Model 80 Stepping 0, AuthenticAMD`
- Timing source: `run_metadata.json` generated by `src.cli.run_pipeline`
- Memory source: peak traced memory recorded in `run_metadata.json`

## 2. Exact Commands Used

Synthetic reproducibility check:

```powershell
python scripts\run_repro_check.py
```

Public-fixture spot benchmarks:

```powershell
python -m src.cli --rdml data\raw\stepone_std.rdml --outdir outputs\bench_stepone --min-cycles 25 --plate-schema 96
python -m src.cli --rdml data\raw\BioRad_qPCR_melt.rdml --outdir outputs\bench_biorad --min-cycles 25 --plate-schema 96
python -m src.cli --rdml data\raw\lc96_bACTXY.rdml --outdir outputs\bench_lc96 --min-cycles 25 --plate-schema 96
```

## 3. Deterministic Reproducibility Check

Artifact match across repeated synthetic runs:

- `well_calls.csv`: match
- `rerun_manifest.csv`: match
- `plate_qc_summary.json`: match after normalizing generated timestamp
- `run_metadata.json`: match after normalizing execution date and runtime

## 5. Interpretation

- The parser and pipeline now run successfully on three public RDML fixtures from the official `PCRuniversum/RDML` example set.
- The current implementation remains comfortably sub-second on the smaller public files and roughly `1.23` seconds on the largest added fixture (`19200` rows).
- `lc96_bACTXY.rdml` currently yields all-review output, which is operationally useful as a stress test but should not be overinterpreted as assay truth.

## 6. Limits

- These timings were collected on a single Windows development machine and are not cross-platform benchmarks.
- Public-fixture outcomes are not ground-truth clinical labels; they show parser coverage and pipeline throughput, not assay correctness.
- Peak traced memory is now recorded, but it reflects Python-level traced allocations rather than full process RSS.
